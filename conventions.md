# Конвенции проекта

1. **Чтение-запись в БД не обкладывается `try-except`.** Исключения обрабатываются выше — в главном цикле и в каждом потоке раннером.

2. **Единый способ логирования — `write_log` и `write_log_entry`.** Запись в БД-логи напрямую запрещена. Прямой вызов `print` запрещён по всему проекту. Только `write_log_entry`! Единственное исключение — сам `write_log_entry`, где `print` используется внутри реализации.

3. **Переменные окружения читаются из базы только в `refresh_environment()` и передаются только через `EnvSnapshot`.** Никакого прямого чтения из базы по всему остальному коду.

4. **`notify_failure` сама владеет своим `try-except`.** `write_log_entry` вызывает её напрямую, без обёртки. Никто снаружи не оборачивает `notify_failure` в `try-except`.

5. **Два типа исключений для разных сценариев.** `AppException` — для прерывания пайплайна по ожидаемой причине. `FatalError` — для нарушения инварианта, то есть «этого не должно было произойти никогда».

6. **UUID и ID в сообщениях лога — только через `fmt_id_msg`.** Прямая вставка идентификаторов в строку запрещена.

7. **Потоки читают окружение только через снимок, захваченный один раз.** `snap = environment.snapshot()` — первая строка `run()`. Обращение к `environment.*` напрямую внутри потока запрещено.
